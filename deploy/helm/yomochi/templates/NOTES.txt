────────────────────────────────────────────────────────────────────────────
INSTALL ORDERING — important
────────────────────────────────────────────────────────────────────────────

The chart expects the following to exist BEFORE `helm install`:

  - Secret "{{ .Values.existingSecret }}" with keys: DATABASE_URL, REDIS_URL,
    KAFKA_BOOTSTRAP_SERVERS, OPENAI_API_KEY, JWT_SECRET.
  - Postgres reachable from the cluster (DATABASE_URL).

Order Helm follows on install / upgrade:

  pre-install hook  →  migration Job  (must succeed)
                    ↓
  rendered resources →  api + 5 workers + web + ingress + PodMonitor

Use `--atomic --wait` on `helm upgrade --install` so a failed hook rolls the
release back to the previous revision instead of leaving you with a half-deploy.
